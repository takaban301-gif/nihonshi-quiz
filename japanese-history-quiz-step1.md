# 日本史問題演習アプリ 開発指示書(第1段階)

## このドキュメントについて

これは Claude Code に渡す**第1段階の開発指示書**です。
最終的にスマホで使える日本史の問題演習アプリを作りますが、今回はまず**「1問だけ表示されて、選択肢を選ぶと正誤が出る」**最小バージョンを作ります。

使い方:
1. 任意の作業フォルダを作る (例: `~/projects/nihonshi-quiz`)
2. そのフォルダに移動して `claude` コマンドを実行
3. このドキュメントの内容をそのままClaude Codeに貼り付ける(または「このファイルを読んで指示通りに作って」と伝える)

---

## プロジェクト概要

- **目的**: 大学受験向けの日本史問題演習アプリを開発する
- **利用環境**: スマートフォンのブラウザで利用する(将来的にホーム画面に追加してアプリのように使う想定)
- **方針**: バックエンド/サーバーは持たない。静的Webアプリとして構築する
- **データ管理**: 問題はJSONファイル、ユーザーの進捗はブラウザのlocalStorageを使用する
- **公開先**: 将来的にVercelにデプロイする予定(今回はローカルで動けばOK)

---

## 今回のゴール(第1段階のスコープ)

以下の動作ができれば完了とします。これ以上の機能は**今回は実装しないでください**。

1. ブラウザでアプリを開くと、日本史の問題が**1問だけ**表示される
2. 4つの選択肢が表示される
3. 選択肢をタップ/クリックすると、正解か不正解が分かる
4. 正解の場合は正解と短い解説が表示される
5. 不正解の場合は正しい答えと短い解説が表示される
6. 「次の問題へ」ボタンを押すとリセットされ、また同じ問題(または別の問題)が表示される
7. スマートフォンのブラウザで開いても、文字や選択肢が読みやすく表示される(レスポンシブ対応)

## 技術スタック

- **フレームワーク**: Vite + React (JSX。TypeScriptは使わない)
- **スタイリング**: プレーンなCSS(Tailwind等のフレームワークは使わない)
- **言語**: JavaScript (ES Modules)
- **データ**: ローカルのJSONファイル
- **状態管理**: React の useState のみ(Reduxなどは使わない)

シンプルさを優先してください。余計なライブラリは追加しないでください。

以下の手順で実行してください。


## 問題データの仕様

`src/data/questions.json` に以下の形式で**3問だけ**サンプルデータを作成してください。
データ構造は将来拡張する想定なので、以下のフィールドを必ず含めてください。

```json
[
  {
    "id": "q001",
    "era": "古代",
    "category": "政治",
    "difficulty": 2,
    "question": "710年に元明天皇によって遷都された、奈良時代の都の名称はどれか。",
    "choices": [
      "藤原京",
      "平城京",
      "長岡京",
      "平安京"
    ],
    "answerIndex": 1,
    "explanation": "710年、元明天皇により藤原京から平城京(現在の奈良市)に遷都された。平城京は唐の長安をモデルに造られた。"
  },
  {
    "id": "q002",
    "era": "中世",
    "category": "政治",
    "difficulty": 2,
    "question": "1192年に源頼朝が朝廷から任命された役職はどれか。",
    "choices": [
      "関白",
      "太政大臣",
      "征夷大将軍",
      "摂政"
    ],
    "answerIndex": 2,
    "explanation": "源頼朝は1192年に征夷大将軍に任命され、鎌倉幕府が名実ともに成立した。"
  },
  {
    "id": "q003",
    "era": "近世",
    "category": "政治",
    "difficulty": 3,
    "question": "江戸幕府が大名を統制するために定めた、参勤交代を制度化した法令はどれか。",
    "choices": [
      "禁中並公家諸法度",
      "武家諸法度",
      "御成敗式目",
      "公事方御定書"
    ],
    "answerIndex": 1,
    "explanation": "武家諸法度は1615年に最初に発布され、3代将軍徳川家光が1635年の改訂で参勤交代を制度化した。"
  }
]
```

**フィールドの意味:**

- `id`: 問題の一意なID(将来の進捗管理で使う)
- `era`: 時代区分(古代/中世/近世/近代/現代 など)
- `category`: 分野(政治/経済/文化/外交 など)
- `difficulty`: 難易度(1〜5の整数)
- `question`: 問題文
- `choices`: 選択肢の配列(必ず4つ)
- `answerIndex`: 正解の選択肢のインデックス(0始まり)
- `explanation`: 解説文(短く)

## 機能仕様(詳細)

### App.jsx の役割

- `questions.json` をimportする
- 表示する問題を1つ選ぶ(今回は配列の最初の問題で固定でよい。あるいはランダムでもよい)
- `<QuestionCard>` に問題データを渡す

### QuestionCard.jsx の役割

- props で問題データを受け取る
- 問題文と4つの選択肢ボタンを表示する
- ユーザーが選択肢をクリックしたら:
  - 選んだ選択肢を内部状態(useState)に保持
  - 正誤判定の結果を表示する領域を出す
  - 正解なら緑系、不正解なら赤系の色で結果表示
  - 解説を表示する
  - 選択済みの状態では選択肢ボタンは無効化(再クリックできない)
- 「次の問題へ」ボタンを表示する
  - クリックすると選択状態がリセットされ、初期状態に戻る
  - (今回はデータが1問固定なので、見た目上は同じ問題が再表示されてOK)

## UI / デザイン要件

スマートフォンで快適に使うための最低条件:

- **モバイルファースト**で設計する(画面幅375px程度を基準)
- **タップしやすいボタンサイズ**: 選択肢ボタンは縦に並べ、高さは最低でも48px、左右に余白を取る
- **読みやすいフォントサイズ**: 問題文は16px〜18px、選択肢は16px以上
- **コントラスト**を確保する(白背景に濃いグレーの文字など)
- 横幅は `max-width: 600px` 程度にして、PC で開いたときに広がりすぎないようにする
- viewport の meta タグを `index.html` に必ず入れる: `<meta name="viewport" content="width=device-width, initial-scale=1.0">`(Viteのテンプレートには既に入っているはず)

色味は派手にせず、学習に集中できるシンプルな配色にしてください(白〜薄グレー基調を想定)。
